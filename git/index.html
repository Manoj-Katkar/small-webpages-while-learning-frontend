Comprehensive Guide to Git Commands

Introduction to Git

Git is a distributed version control system used by developers to track changes in source code. It allows multiple developers to work on a project efficiently and keeps track of modifications, additions, and deletions.

Basic Git Commands

Initializing a Repository

git init
Repository 
git remote add origin <repository_url>
git remote -v

Fetching and Checking Out the Main Branch

git fetch origin
git checkout main

Checking Status and Staging Changes

git status
git add .

Committing and Pushing Changes

git commit -m "Your commit message"
git push origin main

Configuring Git User Information

git config user.name
git config user.email

Common Terminal Commands for File Management

mkdir folderName
ls

Viewing Changes and Commit History

git diff
git log

Reverting Changes to the Last Commit

Steps to Revert to a Previous Commit

git log --oneline
git reset --hard <commit_id>

Branching and Merging

Creating a New Branch

git checkout -b <new_branch_name>
git push --set-upstream origin <new_branch_name>

Merging Branches

When you create a pull request, the senior developer or tech lead will review and merge the branch into the main branch.

git checkout main
git pull origin main

Branch Naming Conventions

Prefix Purpose Example
wip Work in progress wip/new-ui-design
feat Adding or expanding a feature feat/chat-support
bug Fixing a bug bug/fix-login-error
junk Experimental changes junk/test-feature

Merge vs. Rebase

Merging is the preferred approach in companies since it keeps a cleaner commit history.

Rebasing is used in special cases where organizations need to track all individual commits made by team members.

Common Git Commands Used in Organizations

git add .

Conclusion

Git is an essential tool for version control, ensuring efficient collaboration and tracking of source code changes. Mastering these Git commands will streamline your development workflow and help in better project management.
